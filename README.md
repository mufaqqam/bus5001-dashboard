BUS5001 — Council Incident Management Dashboard
Live Dashboard: https://sacouncilincidents01.z44.web.core.windows.net/
A static public dashboard for a local council's community incident management platform, built as part of BUS5001 Cloud Platforms and Analytics (Assessment 1).
---
Overview
This repository contains the source code for the council incident dashboard. The dashboard reads a curated JSON summary file generated by an Azure Logic App and presents it as a public-facing web page — no server, no backend, no build tools required.
The full solution consists of:
Azure Blob Storage — three containers for raw, processed, and public data
Azure Logic App — automated daily pipeline that validates, transforms, and archives incoming CSV files
This dashboard — static HTML file served from Azure Blob Storage's static website hosting

Dashboard Features
KPI Cards — Total incidents, open incidents, in-progress, resolved count, and resolution rate
Incidents by Category — Bar chart rendered with Chart.js
Incidents by Priority — Visual breakdown of High, Medium, and Low priority counts
Recent Incidents Table — 10 most recent records with colour-coded status and priority badges
Last Updated Timestamp — Sourced directly from the data file so users know when it was last refreshed

How It Works
The dashboard fetches a pre-aggregated `summary.json` file from the Azure Blob Storage public container on every page load:
```
https://sacouncilincidents01.blob.core.windows.net/public-data/summary.json
```
This file is generated automatically by the Azure Logic App (`logic-council-incidents-proc`) each night at 23:55 AEST. When council staff upload a new incident CSV to the `raw-data` container, the Logic App:
Reads and validates the file
Composes a curated JSON summary with KPIs and recent records
Writes `summary.json` to the `public-data` container
Archives the processed CSV to `processed-data`
Deletes the original from `raw-data`
The dashboard reflects the updated data on the next page load — no manual steps required.

Deployment
The dashboard is deployed via Azure Blob Storage static website hosting. The `index.html` file is uploaded directly to the `$web` container on the `sacouncilincidents01` storage account.
To update the dashboard:
Edit `index.html`
Go to Azure portal → `sacouncilincidents01` → Containers → `$web`
Delete the existing `index.html`
Upload the new `index.html`
The live URL remains unchanged.
> **Note:** Azure Static Web Apps with GitHub Actions CI/CD was the original intended deployment method. It was unavailable under the university student subscription due to regional policy restrictions. Blob static website hosting was used as the alternative.
